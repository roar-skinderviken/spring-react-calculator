My Server Setup

Follow these steps to set up your own server.

Server Setup Guide

Step 1: Raspberry Pi 5 Setup

Setting up Raspberry Pi 5 with Linux and booting it.

  • Gather the necessary components: Raspberry Pi 5, Micro SD card, Micro SD card reader, and micro HDMI cable. Note that if you're able to connect to the internet provider, there won't be any need for a micro HDMI.
  • Load a 64-bit lightweight Linux distribution (e.g., Debian GNU/Linux) onto the Raspberry Pi.
  • This computer will be accessible via the SSH command later on.

Step 2: Setting up NGINX

Setting up the NGINX server to run your home site.

  1. NGINX will serve as the primary server for vicx.no.
  2. Install NGINX using the terminal and configure it via "/etc/nginx/sites-available/default". Choose the appropriate ports and link the server to the domain (domain purchase will come in step 4).
  3. Once configured, edit the HTML files in the "/var/www/html" directory, where the server will initially read the index.html by default.

Step 3: Configure Your Network

Port forwarding and network configuration.

  • Log in to your network, typically using a 192.168.x.x address, and authenticate as the administrator.
  • Configure your router to forward relevant ports to your server.
  • Assign a static IP address to the Raspberry Pi and set up port forwarding for the device.

Step 4: Domain Configuration

Setting up your domain purchased from one.com.

  • Once you have purchased the domain without server hosting, you are ready to proceed.
  • Add your public IP address to ensure that one.com can route the traffic to your server.
  • Here Certbot is used. Certbot automatically sets up and renews SSL/TLS certificates for your site, enabling secure HTTPS connections.
  • Disclaimer: Domains are typically rented, with a renewal period often set at 12 months before a new payment is required.

Step 5: Setting up a Java Server

Setting up Apache Tomcat to run Java applications.

  • Download and start Apache Tomcat. Ensure you have the ability to create .war files.
  • Configure NGINX to serve Tomcat on port 8080 and deploy the .war file under "/opt/tomcat/webapps". Start the server with "sudo /opt/tomcat/bin/startup.sh".
  • Create the .war file using Spring Boot and transfer it to the Raspberry Pi using the scp command: "scp /path/to/file username@server:/destination/path".

Step 6: Setting up JavaScript server

Setting up Node.js to run JavaScript applications.

  • Download Node.js as well as npm, we will host our server via pm2.
  • Create the JavaScript file, in this instance create a "Hello World" to see if it runs.
  • pm2 will start our servers, example of execution: "pm2 start helloworld.js".
  • Note that if you have multiple servers, pm2 will only restart one at a time. To properly restart pm2 run "pm2 restart all".
